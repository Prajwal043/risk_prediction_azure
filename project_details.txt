# Intelligent Change Risk Prediction System

### End-to-End Data Science + MLOps + Full Stack Project

---

# 1. Business Problem

### Problem Statement

In software companies, deployments frequently fail due to:

* Large number of files changed
* Too many dependencies
* Poor testing
* Inexperienced engineers
* Night-time deployments
* Database changes

A failed deployment can result in:

* Service outages
* Revenue loss
* Customer dissatisfaction
* Emergency rollbacks

### Goal

Build an AI system that predicts:

> "Will this deployment fail before it goes live?"

and provide:

* Failure Probability
* Risk Level
* Risk Factors
* Explainability

# 3. Dataset Creation

Since real deployment data was unavailable:

### Synthetic Dataset Generated

Rows:

```text

```

Features:

| Feature             | Description               |
| ------------------- | ------------------------- |
| systems_impacted    | Systems affected          |
| num_files_changed   | Number of files modified  |
| deployment_hour     | Deployment time           |
| deployment_day      | Deployment day            |
| engineer_experience | Experience in years       |
| historical_failures | Previous failures         |
| dependency_count    | Dependencies impacted     |
| rollback_required   | Previous rollback history |
| test_coverage       | Test coverage percentage  |
| change_type         | Database/App/Config etc   |

Target:

```text
change_failed

0 = Success
1 = Failure
```

# 4. Failure Logic

Synthetic risk formula:

```python
risk_score = (
    dependency_count * 2
    + historical_failures * 3
    + systems_impacted
)
```

Additional risk:

* Low test coverage
* Night deployment
* Weekend deployment
* Database change
* Junior engineer

Converted to:

```text
Failure Probability
```

using sigmoid function.

# 2-Minute Interview Summary

> I built an end-to-end Intelligent Change Risk Prediction system that predicts whether a software deployment is likely to fail before it is released. Since real deployment data wasn't available, I generated a synthetic dataset of 10,000 deployment records using realistic risk patterns. I performed EDA, engineered deployment-specific features like complexity score, after-hours flag, and weekend deployment indicators, and built multiple ML models including Logistic Regression, Random Forest, and XGBoost. After hyperparameter tuning, XGBoost achieved the best ROC-AUC and was selected as the final model.
>
> I used SHAP for explainability, allowing both global and deployment-level risk explanations. The model was tracked using MLflow, exposed through a FastAPI REST API, persisted predictions in PostgreSQL, and integrated with a React dashboard for visualization and risk assessment. The entire solution was containerized using Docker and designed for CI/CD with GitHub Actions and cloud deployment on Azure App Service with Azure Blob Storage and PostgreSQL. This project demonstrates the complete ML lifecycle from data generation and modeling to deployment, monitoring, and MLOps.
